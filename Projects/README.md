## Project Details

### 1. Calculator (`01_calculator/`)

**What you'll build:** A command-line calculator with basic operations and history tracking.

**Features:**
- Basic operations: add, subtract, multiply, divide
- Operation history
- Input validation and error handling
- User-friendly interface

**What you'll learn:**
- Functions and control flow
- Input validation
- Error handling with try/except
- User interaction design

---

### 2. Todo App (`02_todo_app/`)

**What you'll build:** A complete todo list application with file persistence.

**Features:**
- Add, edit, delete, and mark tasks as complete
- Save tasks to file and load on startup
- Priority levels and due dates
- Search and filter functionality

**What you'll learn:**
- Object-oriented programming (classes, methods)
- File I/O for data persistence
- Data structures for task management
- User interface design

---

### 3. File Organizer (`03_file_organizer/`)

**What you'll build:** A utility to automatically organize files by type.

**Features:**
- Sort files into folders by extension
- Batch processing of multiple files
- Logging of operations
- Custom organization rules

**What you'll learn:**
- File system operations and pathlib
- Error handling for file operations
- Logging and debugging
- Working with the OS module

---

### 4. Data Analyzer (`04_data_analyzer/`)

**What you'll build:** A tool to analyze and visualize data from CSV and JSON files.

**Features:**
- Load datasets from CSV/JSON
- Calculate statistics (mean, median, mode, etc.)
- Filter and sort data
- Generate reports and summaries

**What you'll learn:**
- Data structures and algorithms
- File I/O for data processing
- Statistical calculations
- Data visualization concepts

## Project Workflow

For each project, follow this approach:

1. **Phase 1: Planning** (30 minutes)
   - Read the requirements carefully
   - Design your data structures and functions
   - Write pseudocode for key features

2. **Phase 2: Implementation** (2-4 hours)
   - Build core functionality first
   - Add error handling
   - Implement user interface

3. **Phase 3: Testing** (30 minutes)
   - Test with various inputs
   - Handle edge cases
   - Fix bugs

4. **Phase 4: Enhancement** (optional)
   - Complete challenge tasks
   - Refactor code
   - Add your own features
