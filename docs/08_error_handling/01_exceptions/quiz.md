## Question 1
What is the purpose of an exception?

- A) Improve performance
- B) Signal an error and stop incorrect execution
- C) Replace functions
- D) Hide bugs

<details>
<summary>Show Answer</summary>

**B)** Exceptions signal failures and stop invalid execution paths.

</details>

## Question 2
Which exception is raised by `int("abc")`?

- A) TypeError
- B) ValueError
- C) KeyError
- D) IndexError

<details>
<summary>Show Answer</summary>

**B)** Invalid literal conversion raises `ValueError`.

</details>

## Question 3
What does a stack trace show?

- A) The list of variables in scope
- B) The sequence of calls leading to an error
- C) The memory layout of a program
- D) The time complexity of functions

<details>
<summary>Show Answer</summary>

**B)** A stack trace shows the call path to the error.

</details>

## Question 4
When should you raise your own exceptions?

- A) Never
- B) Only in libraries
- C) When input violates required rules
- D) Only for type errors

<details>
<summary>Show Answer</summary>

**C)** Raise errors when rules or invariants are broken.

</details>
