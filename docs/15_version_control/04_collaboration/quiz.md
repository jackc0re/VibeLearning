## Question 1
What does `git clone` do?

- A) Copies the entire Git repository to your local machine
- B) Creates a new empty repository
- C) Downloads only the latest commit
- D) Backs up your files

<details>
<summary>Show Answer</summary>

**A)** `git clone` creates a complete copy of a remote repository on your local machine, including all history, branches, and tags.

</details>

## Question 2
What is the difference between `git fetch` and `git pull`?

- A) They are the same
- B) `fetch` downloads, `pull` downloads and merges
- C) `pull` downloads, `fetch` downloads and merges
- D) `fetch` works with SSH, `pull` works with HTTPS

<details>
<summary>Show Answer</summary>

**B)** `git fetch` downloads changes from the remote but doesn't merge them. `git pull` does both: it fetches and then merges the changes into your current branch.

</details>

## Question 3
What does `git push` do?

- A) Downloads changes from remote
- B) Uploads your local commits to a remote repository
- C) Creates a new repository
- D) Deletes remote files

<details>
<summary>Show Answer</summary>

**B)** `git push` uploads your local commits to the remote repository, making your changes available to others.

## Question 4
What is a remote in Git?

- A) A different branch in the same repository
- B) A version of your project hosted elsewhere
- C) A backup of your files
- D) A merge conflict

<details>
<summary>Show Answer</summary>

**B)** A remote is a version of your repository hosted on another server, such as GitHub, GitLab, or a company's internal server.

</details>

## Question 5
What does `git push -u origin feature` do?

- A) Deletes the feature branch
- B) Pushes and sets the upstream branch
- C) Updates only the origin remote
- D) Creates a new repository

<details>
<summary>Show Answer</summary>

**B)** The `-u` flag sets the upstream branch, meaning future `git push` and `git pull` commands will use this remote/branch as the default.

</details>

- C) A command to download files

<details>

## Question 9
When should you use `--force` with `git push`?

- A) Every time you push
- B) When you want to overwrite remote history (use carefully!)
- C) When pulling changes
- D) When cloning a repository

<details>
<summary>Show Answer</summary>

**B)** `git push --force` overwrites the remote repository's history. Use with caution! `--force-with-lease` is safer as it checks if the remote has changed.

</details>

## Question 10
What does `git branch -r` show?

- A) Only local branches
- B) Remote branches
- C) All branches
- D) Only the main branch

<details>
<summary>Show Answer</summary>

</details>

## Question 11
What is the difference between SSH and HTTPS for Git?

- A) SSH uses tokens, HTTPS uses keys
- B) HTTPS uses tokens/passwords, SSH uses keys
- C) They are identical
- D) HTTPS doesn't work with Git

<details>
<summary>Show Answer</summary>

**B)** HTTPS authenticates with username/password or tokens, while SSH uses SSH key pairs. SSH is generally preferred for frequent operations.

</details>

## Question 17
What command shows commits that are in local but not on remote?

- A) `git log origin/main..main`
- B) `git log main..origin/main`
- C) `git log --remote`
- D) `git log --all`

<details>
<summary>Show Answer</summary>

## Question 18
Why would you use `git pull --rebase`?

- A) To download without merging
- B) To create a linear history instead of merge commits
- C) To push changes
- D) To delete branches

<details>
<summary>Show Answer</summary>

**B)** `--rebase` repositions your local commits on top of the fetched changes, creating a linear history instead of a merge commit.

</details>
