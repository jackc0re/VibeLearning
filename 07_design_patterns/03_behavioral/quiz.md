## Question 1
What does the Observer pattern enable?

- A) One object notified by many subjects
- B) Many observers notified when a subject changes
- C) Global access to a single instance
- D) Automatic caching

<details>
<summary>Show Answer</summary>

**B)** Observers subscribe to updates from a subject.

</details>

## Question 2
Which pattern encapsulates interchangeable algorithms?

- A) Strategy
- B) Adapter
- C) Facade
- D) Builder

<details>
<summary>Show Answer</summary>

**A)** Strategy lets you swap algorithms without changing callers.

</details>

## Question 3
What is a common use for the Command pattern?

- A) Building objects step-by-step
- B) Wrapping incompatible interfaces
- C) Queuing, logging, or undoing actions
- D) Enforcing a single instance

<details>
<summary>Show Answer</summary>

**C)** Commands encapsulate actions for history or delayed execution.

</details>
